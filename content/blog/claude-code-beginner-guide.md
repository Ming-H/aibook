## 第一章：为什么你应该试试 Claude Code

你有没有想过，有一天写代码不用自己敲键盘？

说起来，这不是科幻。Claude Code 就是这么个东西——你用自然语言告诉它想做什么，它帮你把代码写出来。

跟你在网页上用 ChatGPT 不一样，Claude Code 直接在你电脑的终端里跑。它能读取你的项目文件，理解你的代码结构，然后直接改代码、跑命令。说白了，它就像一个坐你旁边的程序员同事，你说需求，它干活。

很多人觉得 AI 编程离自己很远，或者觉得自己不懂技术用不了。其实恰恰相反——越是新手，越能感受到效率提升。以前查一个 bug 要半天，现在描述一下问题，AI 帮你定位。以前写一个小功能要踩无数坑，现在让 AI 先写个初稿，你再微调。

还有些人担心：用了 AI 是不是就学不会编程了？老实说，恰恰相反。AI 能帮你理解代码在做什么，你问它"这段代码什么意思"，它会解释给你听。比一个人闷头查文档效率高多了。

Claude Code 是 Anthropic 官方出的工具，不是什么野路子。安装简单，用起来也直观，特别适合想尝试 AI 编程的新手。而且它免费额度足够个人使用，先试试又不花钱。

## 第二章：安装 Claude Code，5 分钟搞定

安装前先确认一件事：你电脑上有 Node.js 吗？

打开终端（Windows 用 PowerShell），输入 `node --version`。如果显示版本号，比如 v20.x.x，那就没问题。如果提示"命令未找到"，去 nodejs.org 下载安装一个，选 LTS 版本就行。

有了 Node.js，安装 Claude Code 就一句话：

```
npm install -g @anthropic-ai/claude-code
```

Mac 用户也可以用 Homebrew：`brew install --cask claude-code`

Windows 用户还能用 WinGet：`winget install Anthropic.ClaudeCode`

装完之后，输入 `claude --version` 确认一下。看到版本号就说明安装成功了。

第一次运行 `claude` 命令时，它会让你登录。用你的 Claude 账号授权一下就行，跟登录其他软件没什么区别。

## 第三章：第一次对话——让 AI 理解你的项目

安装好了，我们来试试。

先 cd 到你的项目目录，然后输入 `claude`。你会看到一个 `>` 提示符，这就意味着 Claude 准备好听你指挥了。

第一件事，让它先熟悉一下你的项目。输入：

```
帮我看看这个项目的结构，告诉我它是做什么的
```

Claude 会自动读取你的文件，分析项目结构,然后给你一个总结。它会告诉你这是什么类型的项目、用了什么技术栈、入口文件在哪里。

有意思的是，它不只是简单列文件。它会理解代码之间的关系，告诉你"这个文件调用了那个文件"、"这里是配置"、"那里是业务逻辑"。

如果你没有项目，想从零开始，也可以直接说：

```
帮我创建一个简单的个人名片网页，用 HTML 和 CSS
```

它就会帮你创建文件、写代码，一步一步来。

## 第四章：核心命令，这几个最常用

Claude Code 有一些以 `/` 开头的命令，记住了能省不少事。

`/help` - 忘记怎么用了？输入这个看看帮助。

`/clear` - 想开始一个新任务，不想被之前的对话干扰？清空历史重新来。

`/compact` - 这个很重要。聊久了对话会变得很长，占用很多 token。用这个命令可以"压缩"上下文，保留关键信息但释放空间。

`/cost` - 看看这次对话花了多少 token。如果你关心成本，偶尔看看这个。

`/doctor` - 遇到莫名其妙的问题？运行诊断看看是不是环境配置有问题。

`/status` - 查看当前配置状态，用的什么模型、API 地址是什么。

除了这些斜杠命令，你还可以直接让它执行系统命令。比如：

```
运行 npm install
```

```
执行 npm run build
```

它会帮你跑命令，如果有报错还会帮你分析原因。

## 第五章：写一个完整的小项目

光说不练假把式。我们来写一个简单的个人名片网页。

在空目录里启动 Claude Code，然后输入：

```
帮我创建一个个人名片网页。要求：
1. 一个 index.html 文件
2. 包含名字、一句话介绍、三个社交链接
3. 用内联 CSS 做一个简洁的卡片样式
4. 居中显示，背景用浅灰色
```

Claude 会帮你创建文件、写代码。写完之后，你可以说：

```
加一个头像占位符，圆形的
```

它会在现有代码基础上修改。不满意？继续说：

```
把配色改成蓝色系，更专业一点
```

它就会调整 CSS。整个过程中，你不需要写一行代码，只需要用自然语言描述你想要什么。

这就是 Claude Code 最香的地方——你关注的是"想要什么"，而不是"怎么实现"。

## 第六章：常见问题和避坑指南

用了一段时间，你可能会遇到一些问题。

**连不上怎么办？**

检查一下网络。如果你在国内，可能需要配置代理。或者用国内的中转服务，比如阿里云的 Coding Plan。配置方法是在 `~/.claude/settings.json` 里设置 `ANTHROPIC_BASE_URL`。

**Token 消耗太快怎么办？**

对话太长会占用大量 token。定期用 `/compact` 压缩一下。另外，尽量一次说清楚需求，别反复改来改去。

**改错代码怎么办？**

它会改错，别指望 100% 正确。关键是让它解释改动内容，你自己 review。有问题就说"这个不对，应该是xxx"，它会修正。

**能不能让它自动改？**

可以，但要谨慎。简单任务让它直接改，复杂任务先让它说明要改什么，你确认后再执行。

## 第七章：Claude Code vs Cursor vs Copilot

你可能会问：这玩意儿跟 Cursor、Copilot 比怎么样？

说实在的，各有各的场景。

**Cursor** 是个完整的 IDE，基于 VS Code 改的。如果你想要一个开箱即用的编程环境，Cursor 很方便。图形界面，鼠标点点就能用。

**Copilot** 是代码补全工具，集成在各种 IDE 里。它擅长的是"你写一行，它补全下一行"。适合你已经知道要写什么，只是想快一点。

**Claude Code** 是命令行工具。它不依赖特定 IDE，能深度理解你的整个项目。你用自然语言描述需求，它帮你规划和执行。更像是一个"能干活的同事"，而不是"打字加速器"。

如果你是新手，想体验"用自然语言写代码"的感觉，Claude Code 是个很好的起点。如果你已经是老手，想在现有工作流里加点 AI 辅助，Copilot 可能更顺手。

## 第八章：下一步怎么学

学会了基本操作，接下来怎么提升？

**多看官方文档**。Anthropic 的文档写得挺清楚，有很多示例。docs.anthropic.com 上有 Claude Code 的完整说明。官方文档还有一个好处——永远是最新的，网上搜到的教程可能过时了。

**加入社区**。GitHub 上有 Claude Code 的讨论区，遇到问题可以去搜搜看。国内的话，CSDN、掘金上也有不少教程和经验分享。有问题别闷着，问问别人可能一句话就解决了。

**从小项目开始**。别一上来就想用它写大项目。先拿它写点小工具、小页面，熟悉了再逐步扩大使用范围。等你对它的能力边界有感觉了，再挑战复杂任务。

**建立自己的工作流**。每个人的习惯不一样。有些人喜欢先让 AI 分析再让它动手，有些人喜欢直接让它干。找到适合自己的节奏最重要。

**学会写好提示词**。同样一件事，描述方式不同，效果差很多。尽量具体，一次说清楚。"帮我写一个按钮"不如"帮我写一个蓝色的提交按钮，点击后提交表单"效果好。

AI 编程不是要取代程序员，而是让你把精力放在更有价值的事情上。重复性的工作交给 AI，创造性的决策留给自己。

现在就开始试试吧。打开终端，输入 `claude`，然后告诉它你想做什么。记住，它不是完美的，但足够好用。错了就让它改，改到你满意为止。
